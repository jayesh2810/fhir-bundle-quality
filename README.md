# FHIR Bundle Quality Analyzer

A tool for evaluating the quality and consistency of FHIR (Fast Healthcare Interoperability Resources) Bundles.

## What it does
The analyzer evaluates bundles across five key quality dimensions:
- **Completeness**: Checks for the presence of required fields in key resource types (Condition, MedicationRequest, Observation, Encounter).
- **Code Coverage**: Verifies that clinical resources use recognized coding systems (e.g., SNOMED, LOINC, RxNorm).
- **Temporal Consistency**: Detects logical date violations, such as onset dates occurring before a patient's birth date.
- **Duplicate Detection**: Uses LLM-powered embeddings to identify semantically duplicate resources within a bundle.
- **Overall Grade**: Aggregates all scores into a weighted final grade (A-F) to provide a quick quality snapshot.

## Tech Stack
- **Backend**: Python, FastAPI, Pydantic, Google GenAI SDK (for embeddings), Scikit-learn.
- **Frontend**: Svelte, Vite.

## How to Run

### Backend
1. Navigate to the backend directory: `cd backend`
2. Install dependencies: `pip install -r requirements.txt`
3. Set your Google API Key: `export GOOGLE_API_KEY="your_api_key_here"`
4. Start the server: `uvicorn main:app --reload --port 8000`

### Frontend
1. Navigate to the frontend directory: `cd frontend`
2. Install dependencies: `npm install`
3. Start the dev server: `npm run dev`
4. Open `http://localhost:5173` in your browser.
